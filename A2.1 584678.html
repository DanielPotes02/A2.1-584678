A2.1 Regresin logstica y validacin cruzada

En este trabajo, se analizar una base de datos histrica que contiene informacin sobre los pasajeros del Titanic, con el objetivo de predecir la probabilidad de supervivencia de cada uno de ellos utilizando un modelo de regresin logstica. Esta tcnica es adecuada, ya que la variable a predecir ("Survived") es binaria: 1 significa que el pasajero sobrevivi, y 0 que no lo hizo.

Variables utilizadas:

  • Survived: Variable objetivo (0 = No sobrevivi, 1 = Sobrevivi).
  • Pclass: Clase del pasajero (1 = Primera clase, 2 = Segunda clase, 3 = Tercera clase).
  • Age: Edad del pasajero.
  • SibSp: Nmero de hermanos o cnyuges a bordo.
  • Parch: Nmero de padres o hijos a bordo.
  • Fare: Tarifa pagada por el pasajero.
  • Cabin_known: Indica si se conoce el nmero de camarote (1 = S, 0 = No).
  • Sex_male: Gnero del pasajero (1 = Hombre, 0 = Mujer).
  • Embarked_Q: Indica si el pasajero embarc en el puerto Queenstown (1 = S, 0 = No).
  • Embarked_S: Indica si el pasajero embarc en el puerto Southampton (1 = S, 0 = No).
  • Con estas variables, el modelo buscar patrones que permitan identificar qu tipo de pasajeros tuvieron mayor probabilidad de sobrevivir al hundimiento.

Es importante mencionar que dentro de las variables del dataset, la variable Pclass es una variable categrica, ya que representa la clase en la que viajaba el pasajero (Primera, Segunda o Tercera clase). Este tipo de variables no puede ser interpretado correctamente por un modelo de regresin logstica si se trata como una variable numrica ordinal, ya que el modelo podra asumir una relacin lineal entre las clases, lo cual no es adecuado en este caso.

Para resolver este problema, se procedi a realizar una transformacin conocida como codificacin one-hot o creacin de variables dummies, lo que implica convertir cada categora en una nueva columna binaria. De esta forma, el modelo podr interpretar correctamente la pertenencia a cada clase sin inducir relaciones artificiales entre ellas.

Este paso es fundamental para asegurar que el anlisis y las predicciones del modelo no se vean distorsionados por un tratamiento inadecuado de las variables categricas.

Con el objetivo de medir el desempeo inicial del modelo de regresin logstica, se aplicaron tcnicas de validacin cruzada sobre el conjunto de entrenamiento. La validacin cruzada permite obtener una estimacin ms robusta de la exactitud del modelo, al evaluar su desempeo en mltiples particiones de los datos.

Para este anlisis, se utilizaron dos enfoques:

Leave-One-Out Cross Validation (LOOCV): Esta tcnica consiste en entrenar el modelo utilizando todos los datos de entrenamiento excepto una observacin, que se deja fuera y se utiliza como validacin. Este proceso se repite tantas veces como observaciones haya, cambiando en cada iteracin la observacin que se deja fuera. Validacin cruzada K-Fold: En este caso, los datos de entrenamiento se dividieron en K=5 subconjuntos o "folds". El modelo se entrena en 4 de estos subconjuntos y se valida en el restante, repitiendo el proceso hasta que cada fold haya sido utilizado como validacin.

Para evaluar el desempeo del modelo, se definieron tres umbrales de clasificacin diferentes:

  • Umbral 0.5: Es el valor comnmente utilizado en problemas de clasificacin binaria.
  • Umbral 0.75: Exige mayor certeza para predecir que un pasajero sobrevivi.
  • Umbral 0.25: Relaja el criterio para considerar que un pasajero sobrevivi.

Para cada uno de estos umbrales, se generaron las respectivas matrices de confusin, permitiendo calcular las siguientes mtricas clave:

  • Exactitud: Proporcin de predicciones correctas sobre el total de observaciones.
  • Sensibilidad: Proporcin de pasajeros que realmente sobrevivieron y fueron correctamente identificados como sobrevivientes.
  • Especificidad: Proporcin de pasajeros que no sobrevivieron y fueron correctamente identificados como fallecidos.
  • Precisin: Proporcin de predicciones positivas que realmente correspondan a pasajeros que sobrevivieron.
  • F1 Score: Promedio armnico entre precisin y sensibilidad, que permite evaluar el balance entre ambos indicadores.

Este anlisis permite observar cmo cambia el comportamiento del modelo al modificar el umbral de decisin, lo cual es importante para ajustar el modelo dependiendo de si se quiere priorizar la deteccin de sobrevivientes (alta sensibilidad), minimizar los falsos positivos (alta especificidad), o buscar un equilibrio entre ambos (alto F1 Score), segn los objetivos del anlisis.

# Generar matriz de confusio, y exactitud, sensibilidad y especificidad del modelo antes 3 diferentes umbrales (0.5, 0.75, y 0.25)
def confusion_matrix_threshold(y_test, y_pred_prob, threshold):
    y_pred = np.where(y_pred_prob > threshold, 1, 0)
    conf_matrix = confusion_matrix(y_test, y_pred)
    return conf_matrix

thresholds = [0.25, 0.5, 0.75]
for threshold in thresholds:
    print("Matriz de confusin con umbral de probabilidad de ", threshold)
    print(confusion_matrix_threshold(y_test, y_pred_prob, threshold))
    
    y_pred = np.where(y_pred_prob > threshold, 1, 0)
    print("Exactitud: ", accuracy_score(y_test, y_pred))
    print("Sensibilidad: ", recall_score(y_test, y_pred))
    print("Especificidad: ", recall_score(y_test, y_pred, pos_label=0))
    print("Precision: ", precision_score(y_test, y_pred))
    print("F1: ", f1_score(y_test, y_pred))
    print("\n")
